تست تاخیر (پینگ تقریبی)

یک یا چند آدرس (IP یا دامنه) وارد کن، هرکدوم در یک خط. این ابزار زمان رفت‌وبرگشت درخواست HTTP رو اندازه می‌گیره — این پینگ واقعی ICMP نیست، چون مرورگرها به ICMP دسترسی ندارن. برای دامنه‌های دارای وب‌سرور فعال (مثل google.com) نتیجه دقیقه؛ برای IP خامی که سرویس HTTP روش فعال نباشه، ممکنه «بدون پاسخ» نشون بده.
وضعیت آدرس حداقل میانگین حداکثر Loss
یادآوری: اعداد این جدول بر اساس زمان پاسخ HTTP/HTTPS اندازه‌گیری می‌شن، نه بسته‌ی ICMP خام. اگر روی یک IP «بدون پاسخ» گرفتی، لزوماً یعنی آن سرور آفلاینه — ممکنه فقط پورت وب (80/443) روش باز نباشه. برای پینگ ICMP واقعی و صددرصد دقیق، لازمه از ابزار سیستم‌عامل (مثل دستور ping در ترمینال) یا یک برنامه‌ی بک‌اند استفاده کنی.
